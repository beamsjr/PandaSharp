# PandaSharp Standards Profile

## Enforce
- **Nullable Reference Types** — `<Nullable>enable</Nullable>` in Directory.Build.props
- **Clean Code** — Method length ≤50 lines, guard clauses, VerbNoun naming
- **XML Doc Comments** — All public APIs must have `<summary>` docs
- **Naming Conventions** — PascalCase for public members, camelCase for private, _prefix for fields
- **No Boxing in Hot Paths** — Use typed fast paths (Column<double>.Values) instead of GetObject()
- **Performance** — Use Span<T>/ReadOnlySpan<T> for array access, avoid unnecessary allocations
- **Thread Safety** — Parallel operations must not share mutable state
- **Error Handling** — Validate parameters at public API boundaries, throw meaningful exceptions

## Warn
- **Class Size** — Prefer under 300 lines, flag 500+
- **Constructor Dependencies** — Prefer under 7, flag 10+
- **Test Coverage** — New public APIs should have corresponding tests
- **Records vs Classes** — DTOs and result types should be records
- **Async/Await** — Async all the way, no .Result or .Wait()
- **IDisposable** — Types holding unmanaged resources must implement IDisposable

## Skip
- **Service Layer / Controller Pattern** — Not a web application
- **API Design (REST)** — No HTTP endpoints
- **EF Core / Database** — Handled in separate IO.Database project
- **CQRS** — Not applicable
- **Logging** — Library project, no structured logging requirement
- **Security / JWT** — Not applicable
