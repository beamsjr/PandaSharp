Why PandaSharp Wins on Joins & Slicing

PandaSharp's typed int hash join avoids the generic hash table that pandas uses. The zero-copy Arrow slicing for Head/Tail means no data is copied — just a pointer offset. And compiled C# lambdas run ~8x faster than Python's interpreted apply().

Where Python Wins

pandas' Cython-compiled GroupBy and NumPy vectorized arithmetic are hard to beat from managed code. These operations compile to tight C loops with no overhead, while PandaSharp goes through .NET's JIT-compiled managed code.
